# 文献处理与切分

将用户提供的完整文献（整本书或论文的 MD 文件）自动切分成小块，
为每块生成描述摘要，并更新全局索引，以便后续按需检索。

## 工作流程

### Step 1: 定位原始文献

扫描 `inbox/` 目录，列出所有待处理的 MD 文件。
如果用户指定了具体文件名，直接使用该文件。

如果 `inbox/` 为空，检查 `library/` 根目录是否有待处理的 MD 文件（用户有时会直接放在 library/ 而非 inbox/），如果发现则正常处理，并提示用户下次放入 inbox/。

如果以上均为空且用户提供了文件路径，从该路径读取。

### Step 2: 读取并分析文献结构

读取原始文献内容，分析：
- 文献类型：专著(M)、论文(J)、译著、学位论文(D)等
- 文档结构：标题层级、章节划分、段落组织
- 总字数和预计切分数量
- 引用信息（作者、书名/篇名、出版信息）——通常在文件开头或由用户提供

**检测文件是否为平铺型**：统计 `##` 和 `###` 标题数量，若两者之和少于全部标题数的 10%，判定为"平铺型文件"（通常来自 MinerU 等 PDF 转换工具），需执行 Step 2.5。

向用户简要报告分析结果，确认切分方案后继续。

### Step 2.5: 噪音清洗与标题层级修复（仅平铺型文件）

分两个阶段处理：

#### 阶段一：通用噪音清洗

运行插件内置脚本：
```
python3 ${CLAUDE_PLUGIN_ROOT}/tools/clean-mineru.py <原文件路径> library/<name>/source-cleaned.md
```

脚本完成以下清洗工作：
1. 删除 CDN/HTTP 图片行（`![image](https://...)`）
2. 删除孤立页码行（仅含数字和空格的单独行）
3. LaTeX 还原：`$①$`→`①`、`$[N]$`→`[N]`、`$N\sim M$`→`N–M`、排版规格行删除、TeX 命令剥离（`\mathrm{X}`→`X` 等）、间距数字折叠（`6 5 3`→`653`）、数字间连字符转 en-dash
4. 重复出现 2 次以上的 `#` 级标题只保留第一次
5. 段落合并：修复 PDF 换页导致的段落截断（检测"行尾无句末标点 + 空行 + 小写开头续行"模式并合并）

**注意**：此脚本不做标题层级推断，`#` 标题保持原样。脚注检测和子节标记由 per-book 脚本处理。

#### 阶段二：Per-book 标题层级修复

每本书的标题结构不同，使用 per-book 脚本处理。

**若 `library/<name>/fix-headings.py` 已存在**，直接运行：
```
python3 library/<name>/fix-headings.py library/<name>/source-cleaned.md library/<name>/source-fixed.md
```

**若不存在**，AI 按以下抽样阅读协议分析文献结构，然后编写 `fix-headings.py`：

> **PDF 源文件定位**：优先检查 `inbox/` 或 `library/<name>/` 目录下是否存在同名 PDF 文件。
> 若不存在，询问用户提供 PDF 路径。若用户无法提供 PDF，退回纯 MD 抽样模式（仅执行下方 A 部分）。

> **抽样阅读协议 v2**（MD + PDF 双源对照）：
>
> **A. MD 文本抽样**：
> 1. **提取标题清单**：用 Grep 搜索所有 `^#` 行及行号（通常几十到一两百行，token 极少）
> 2. **首尾各读 200 行**：读取文件开头 200 行和末尾 200 行，了解前言/版权页结构和尾部附录/索引/参考文献结构
> 3. **中间抽样 2-3 段**（每段 50-100 行）：根据标题清单选取代表性位置——选择章节切换处（如某章结尾+下章开头），以确认标题模式在全书一致
>
> **B. PDF 视觉抽样**（3-5 页，≤ 5 页）：
> 4. **首尾各 1 页**：确认版权页/前言结构、尾注/参考文献布局
> 5. **脚注密集页 1-2 页**：选取有多个脚注的页面，确认：
>    - 脚注分隔方式（分隔线? 字号变化? 缩进?）
>    - 脚注编号风格（上标? 方括号? 数字+空格?）
>    - 脚注在 MD 中的对应表现（空行分隔? 无分隔直接拼接?）
> 6. **正文对照 1 页**：选含引用/粗体/斜体的页面，与 MD 对比确认 OCR 系统性错误
>
> **C. 总量控制**：MD ≤ 800 行 + PDF ≤ 5 页
> **D. 判断依据**：标题清单是主要依据（覆盖率 100%），PDF 视觉用于确认脚注格式和 OCR 特征，正文抽样用于确认误标题

分析完成后，编写 `library/<name>/fix-headings.py`，脚本应包含：
- 该书特有的 OCR 修复（如大写标题中的字母间距问题、`I`→`1`/`o`→`0` 等字体混淆）
- 需要删除的行（如 "This page intentionally left blank"、出版商信息伪标题）
- 标题层级分类规则（通常基于精确匹配集合 + 少量正则模式）
- **脚注段落标记**：检测脚注段落（通常为空行后以数字编号开头的段落），在其前插入 `<!-- footnote N -->` 标记行。write-chunks.py 会据此将脚注归集到 chunk 尾部的 Notes 区域。脚注格式因书而异（数字+空格、方括号、圆括号等），需在抽样阅读时确认。
- **子节标记**（如有）：检测章内子节标记（如 `(i)`/`(ii)`/`(iii)`、`(a)`/`(b)` 等行首段落标记），在其前插入 `#####` 标题行，使 plan-chunks.py 能在这些位置切分。子节格式因书而异。

运行脚本后，验证标题层级分布是否合理，必要时微调后重新运行。

修复完成后告知用户 source-fixed.md 已生成，可打开检查；如有误判，用户可手动调整后告知继续。

### Step 3: 确定切分方案

**字符数目标**（中文字符计）：
- 普通目标：800–2500 字
- 强制切分上限：5000 字（超出必须拆）
- 最小合并阈值：400 字（不足则与相邻小节合并）

**切分原则**：
1. 优先以 `##`（章级）为切分边界
2. 某 `##` 节超过上限时，依次在 `###` → `####` → `#####` 边界处切分
3. 仍超出时，在段落末尾（空行处）切分，不在段落中间切断
4. 合并过小节时，优先合并同属一个 `##` 章的相邻小节，不跨章合并

**单篇论文**：按逻辑段落切分（摘要、引言、各节、结论），摘要单独一块。

### Step 3.5: 生成切分草案并审查

直接运行插件内置脚本：
```
python3 ${CLAUDE_PLUGIN_ROOT}/tools/plan-chunks.py library/<name>/source-fixed.md
```

脚本输出 `library/<name>/draft-chunks.md`，格式如下：

```markdown
## 切分草案

chunk-01 [行1-102, 1402字] # 版权信息 · 编者导言
chunk-02 [行103-350, 980字] # 目录
chunk-03 [行351-416, 4322字] ⚠️ 超过2500字 | 候选拆分点：行383（空行）、行401（空行）
chunk-04 [行417-474, 2113字] # 道德形而上学导论·§一
chunk-05 [行475-474, 380字] ↑ 不足400字，建议与相邻块合并
...
```

**AI 审查草案**（读取 draft-chunks.md，不读原文）：
- `⚠️` 标注的大块：判断是否需要拆，选择候选拆分点或指定其他位置
- `↑` 标注的小块：判断与哪个相邻块合并更合逻辑（同章优先）
- 根据标题文字推断论证连贯性（如标题为"§6演绎"紧跟"§6节"，通常应合并）

调整后形成最终切分方案，再执行 Step 4。

### Step 4: 执行切分并保存

1. 为该文献创建目录：`library/<文献简称>/`
   - 文献简称使用英文 kebab-case，如 `kant-metaphysics-morals`、`zhang-rechtstaat-2020`
   - 如果用户通过 `--name` 指定了简称，使用用户指定的

2. 生成 `meta.md`：
```markdown
# <文献标题>

## 引用信息
<GB/T 7714 格式的完整引用>

## 全书/全文概述（200-300字）
<概括文献的核心论点、研究方法、主要结论>

## 与论文的关联
<简述该文献对论文写作的价值和可能的用途>

## 小块清单
共 N 个小块，详见 library/<name>/index.md
```

3. 运行插件内置脚本自动写出所有 chunk 文件：
   ```
   python3 ${CLAUDE_PLUGIN_ROOT}/tools/write-chunks.py \
     library/<name>/source-fixed.md \
     library/<name>/draft-chunks.md \
     library/<name>/ \
     --label "<文献标题>"
   ```
   脚本会自动为每个 chunk 添加来源注释行（格式：`<!-- 来源：<文献标题> | 位置：<章节信息> -->`）。

### Step 5: 为每块生成描述（子 agent 模式）

使用 **Agent 工具**将描述生成委托给子 agent，使其在隔离上下文中完成，避免主上下文积压大量 chunk 内容。

**调用方式**：向子 agent 传入如下任务说明：

> 读取 `library/<name>/` 目录下的所有 chunk 文件（每个使用 `limit: 15` 只读前15行，覆盖来源注释行+标题行+前几行正文即可）。
> 对每个 chunk 生成一条描述，要求：
> - 包含节号范围（如"§53-§56"）
> - 至少1个论证骨架动词："演绎/推导出/证明……"、"区分……与……"、"论证……根据"、"拒绝……而主张……"
> - 2-4个关键术语，附德文原词（若有）
>
> 好的描述示例："法权论导论·§2——区分法权（Recht）概念与道德概念；论证外在强制与普遍自由法则相一致的可能性根据"
> 差的描述（禁止）："第一章的内容"（无节号无动词）、"关于国际法权的讨论"（纯主题词）
>
> 返回格式（纯文本，一行一条，不做其他说明）：
> chunk-01.md: <描述>
> chunk-02.md: <描述>
> ...

**主上下文**收到子 agent 返回的描述列表后，暂存该列表，交由 Step 6.1 写入文件；不在主上下文保留任何 chunk 正文内容。

### Step 6: 更新两级索引

**6.1 新建/更新局部索引** `library/<name>/index.md`

创建该文献的局部索引，包含所有 chunk 的新标准描述：

```markdown
# <文献标题>局部索引

## 引用
<GB/T 7714 格式的简要引用>

## 块描述

共 N 个小块。

- **chunk-01.md**: <Step 5 生成的描述>
- **chunk-02.md**: <描述>
- ...
```

**6.2 更新全局索引** `library/index.md`

在 `library/index.md` 末尾追加书级概述条目（只含一行概述 + 总块数 + 局部索引指针）：

```markdown
## <文献标题>（`<目录名>/`）

[文献类型标记] <简要引用>。<全文一句话概述>。
共 N 个小块 → 详见 `<目录名>/index.md`
```

文献类型标记：`[M]` 专著、`[J]` 期刊论文、`[D]` 学位论文、`[C]` 会议论文、`[N]` 报纸

**全局索引不包含 chunk 级描述列表**——chunk 描述集中在局部索引中管理。

### Step 7: 清理

将处理完的原始文件从 `inbox/` 移到 `inbox/processed/`（自动创建该目录）。
如果原文件在 `library/` 根目录，同样移动到 `inbox/processed/`。

向用户报告处理结果：文献名称、切分块数、索引已更新。

## 批量处理

如果 `inbox/` 中有多个文件，逐个处理，每处理完一个向用户报告。

## 特殊情况

- **用户已按章分好多个 MD**：视为同一文献的多个部分，放入同一个 `library/<name>/` 目录，chunk 编号连续
- **文献缺少引用信息**：询问用户补充，或标注 `[引用信息待补充]`
- **文献内容过短**（不足 400 字）：不切分，整体作为一个 chunk，仍然生成 meta.md 和索引条目
- **用户手动修改了 source-fixed.md**：从修改后的文件重新执行 Step 3 起的流程
