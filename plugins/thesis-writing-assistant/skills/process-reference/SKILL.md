# 文献处理与切分

将用户提供的完整文献（整本书或论文的 MD 文件）自动切分成小块，
为每块生成描述摘要，并更新全局索引，以便后续按需检索。

## 工作流程

### Step 1: 定位原始文献

扫描 `inbox/` 目录，列出所有待处理的 MD 文件。
如果用户指定了具体文件名，直接使用该文件。

如果 `inbox/` 为空，检查 `library/` 根目录是否有待处理的 MD 文件（用户有时会直接放在 library/ 而非 inbox/），如果发现则正常处理，并提示用户下次放入 inbox/。

如果以上均为空且用户提供了文件路径，从该路径读取。

### Step 2: 读取并分析文献结构

读取原始文献内容，分析：
- 文献类型：专著(M)、论文(J)、译著、学位论文(D)等
- 文档结构：标题层级、章节划分、段落组织
- 总字数和预计切分数量
- 引用信息（作者、书名/篇名、出版信息）——通常在文件开头或由用户提供

**检测文件是否为平铺型**：统计 `##` 和 `###` 标题数量，若两者之和少于全部标题数的 10%，判定为"平铺型文件"（通常来自 MinerU 等 PDF 转换工具），需执行 Step 2.5。

向用户简要报告分析结果，确认切分方案后继续。

### Step 2.5: 层级修复与噪音清洗（仅平铺型文件）

检查 `tools/fix-mineru.py` 是否存在：
- 若存在：直接运行 `python3 tools/fix-mineru.py <原文件路径> library/<name>/source-fixed.md`
- 若不存在：生成该脚本并保存到 `tools/fix-mineru.py`，再运行

脚本完成以下工作，结果保存为 `library/<name>/source-fixed.md`：

**噪音清洗**：
1. 删除无效图片行（`![image](https://cdn-mineru...)`、`![image](http...)`）
2. LaTeX 脚注编号还原：`$①$` → `①`，`$[N]$` → `[N]`，`$\d+$` → 纯数字
3. 删除孤立页码行（仅含数字和空格的单独行）
4. 文件开头重复出现的书名行只保留一次

**标题层级推断**（将平铺的 `#` 重写为多级）：

中文学术著作规则：
| 标题内容模式 | 推断层级 |
|------------|---------|
| 第[一二三四五六七八九十百]+部、第[一二…]+卷、书名行（仅含书名） | `#` |
| 第[一二…]+章、×××导论、×××要素论、×××方法论、×××要素学、×××方法学 | `##` |
| 第[一二…]+节、第\d+节、§\d+、一、二、三、（带顿号序号标题） | `###` |
| 第[一二…]+款、决疑论、附释、附录（小节级） | `####` |

英文/德文学术著作规则：
| 标题内容模式 | 推断层级 |
|------------|---------|
| `Part [IVX]+`、`Book [IVX]+`、书名行 | `#` |
| `Chapter \d+`、`Introduction`、`Conclusion`、`Preface`、`Foreword` | `##` |
| `Section \d+`、`§\d+`、罗马数字独立行（`I.`、`II.` 等） | `###` |
| `\d+\.\d+` 形式、`Remark`、`Corollary`、`Note`、`Appendix` | `####` |

无法匹配任何规则的 `#` 标题保持原样。

修复完成后告知用户 source-fixed.md 已生成，可打开检查层级是否正确；
如有误判，用户可手动调整后告知继续。

### Step 3: 确定切分方案

**字符数目标**（中文字符计）：
- 普通目标：800–2500 字
- 强制切分上限：5000 字（超出必须拆）
- 最小合并阈值：400 字（不足则与相邻小节合并）

**切分原则**：
1. 优先以 `##`（章级）为切分边界
2. 某 `##` 节超过上限时，在其内部的 `###` 边界处切分
3. 仍超出时，在段落末尾（空行处）切分，不在段落中间切断
4. 合并过小节时，优先合并同属一个 `##` 章的相邻小节，不跨章合并

**单篇论文**：按逻辑段落切分（摘要、引言、各节、结论），摘要单独一块。

### Step 3.5: 生成切分草案并审查

检查 `tools/plan-chunks.py` 是否存在：
- 若存在：直接运行 `python3 tools/plan-chunks.py library/<name>/source-fixed.md`（或原文件）
- 若不存在：生成该脚本并保存到 `tools/plan-chunks.py`，再运行

脚本输出 `library/<name>/draft-chunks.md`，格式如下：

```markdown
## 切分草案

chunk-01 [行1-102, 1402字] # 版权信息 · 编者导言
chunk-02 [行103-350, 980字] # 目录
chunk-03 [行351-416, 4322字] ⚠️ 超过2500字 | 候选拆分点：行383（空行）、行401（空行）
chunk-04 [行417-474, 2113字] # 道德形而上学导论·§一
chunk-05 [行475-474, 380字] ↑ 不足400字，建议与相邻块合并
...
```

**AI 审查草案**（读取 draft-chunks.md，不读原文）：
- `⚠️` 标注的大块：判断是否需要拆，选择候选拆分点或指定其他位置
- `↑` 标注的小块：判断与哪个相邻块合并更合逻辑（同章优先）
- 根据标题文字推断论证连贯性（如标题为"§6演绎"紧跟"§6节"，通常应合并）

调整后形成最终切分方案，再执行 Step 4。

### Step 4: 执行切分并保存

1. 为该文献创建目录：`library/<文献简称>/`
   - 文献简称使用英文 kebab-case，如 `kant-metaphysics-morals`、`zhang-rechtstaat-2020`
   - 如果用户通过 `--name` 指定了简称，使用用户指定的

2. 生成 `meta.md`：
```markdown
# <文献标题>

## 引用信息
<GB/T 7714 格式的完整引用>

## 全书/全文概述（200-300字）
<概括文献的核心论点、研究方法、主要结论>

## 与论文的关联
<简述该文献对论文写作的价值和可能的用途>

## 小块清单
共 N 个小块，详见 library/index.md
```

3. 用脚本逐块写入 `chunk-01.md`、`chunk-02.md`...，每个 chunk 文件开头加注释：
```markdown
<!-- 来源：<文献标题> | 位置：<章节信息> -->

<原文内容>
```

### Step 5: 为每块生成描述

对每个 chunk，读取**标题行 + 前 5 行非空正文**（而非全文），结合位置信息写描述。

描述要求（60-100字）：
- 该块讨论的**核心主题**
- **关键概念或术语**（便于后续检索匹配）
- 在全文中的**位置/语境**

好的描述示例：
- "法权论导论·§2——法权（Recht）的定义：与义务相关联的强制权限；法权概念与道德概念的区别；外在立法与内在立法的划分"
- "第三章·公法——国家的三权分立原则，立法权、行政权、司法权的康德式论证，权力分立作为法治国家的必要条件"
- "文献综述——国内康德法哲学研究的三个阶段（1980s引介、2000s深化、2010s反思），代表性学者与争议焦点"

差的描述（太模糊）：
- "第一章的内容"
- "关于康德的讨论"

### Step 6: 更新全局索引

读取 `library/index.md`，在末尾追加新文献的索引条目：

```markdown
## <文献标题>（`<目录名>/`）
- **meta.md**: [文献类型标记] <简要引用信息>。<全文一句话概述>
- **chunk-01.md**: <描述>
- **chunk-02.md**: <描述>
- ...
```

文献类型标记：`[M]` 专著、`[J]` 期刊论文、`[D]` 学位论文、`[C]` 会议论文、`[N]` 报纸

### Step 7: 清理

将处理完的原始文件从 `inbox/` 移到 `inbox/processed/`（自动创建该目录）。
如果原文件在 `library/` 根目录，同样移动到 `inbox/processed/`。

向用户报告处理结果：文献名称、切分块数、索引已更新。

## 批量处理

如果 `inbox/` 中有多个文件，逐个处理，每处理完一个向用户报告。

## 特殊情况

- **用户已按章分好多个 MD**：视为同一文献的多个部分，放入同一个 `library/<name>/` 目录，chunk 编号连续
- **文献缺少引用信息**：询问用户补充，或标注 `[引用信息待补充]`
- **文献内容过短**（不足 400 字）：不切分，整体作为一个 chunk，仍然生成 meta.md 和索引条目
- **用户手动修改了 source-fixed.md**：从修改后的文件重新执行 Step 3 起的流程
