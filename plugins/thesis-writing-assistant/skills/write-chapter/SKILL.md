# 章节撰写

撰写论文的具体章节内容。严格按以下流程执行，确保每段论述有文献支撑。

## 工作流程（严格按顺序执行）

### Step 1: 确定写作范围

解析用户请求，确定：
- 要写的章节/小节编号或名称
- 写作模式：
  - **draft**（默认）：逐节撰写初稿。每次只写一个小节（如 2.1、3.2），写完后暂停，等待用户审阅反馈后再继续下一节。若用户指定了具体小节（如"写 2.1"），只写该节；若用户指定了章（如"写第二章"），从该章第一个未完成的小节开始。
  - **batch**：一次性生成完整章节初稿（仅在并行写作模式或用户明确要求"直接写完整章""一口气写完"时使用）
  - **expand**：在已有内容基础上扩展某一节
  - **revise**：根据用户反馈修改已有内容

### Step 2: 读取大纲

读取 `thesis/outline.md`，找到目标章节的大纲内容：
- 该章节的核心论点
- 在全文论证中的位置和作用
- 与前后章节的逻辑关系
- 大纲中标注的预计字数

如果大纲不存在，提示用户先运行 `/thesis-outline`。

### Step 3: 加载参考文献

**优先路径——使用写作计划（推荐，省 token）**：

检查 `thesis/writing-plan.md` 是否存在。如果存在：
1. 读取 writing-plan.md，找到当前章节/小节对应的 chunk 列表
2. **直接读取**列出的 chunk 文件（可并行发出多个 Read 调用）
3. 跳过索引遍历，不需要读 `library/index.md` 或各书的 `index.md`
4. 如果 writing-plan 标注某节"未覆盖"，告知用户文献不足

**回退路径——两级检索（writing-plan 不存在时）**：

1. 读取 `library/index.md` → 确认与当前章节相关的书目
2. 读取相关书目的 `library/<书目目录>/index.md` → 从 chunk 描述中筛选与当前写作内容相关的块
3. **只读取**筛选出的 chunk 文件，不要读取无关的小块
4. 如果某个相关文献的 meta.md 有助于理解全局，也可以读取
5. 如果索引中找不到足够的相关文献，告知用户哪些方面的文献不足

**两条路径均禁止**：一次性读取所有小块文件或整个 library/ 目录。

### Step 4: 读取已有内容

检查 `thesis/` 下是否已有该章节文件。
如有，读取现有内容以保持连贯性和避免重复。

也检查前一章节的结尾段落，确保衔接自然。

**draft 模式续写检测**：如果章节文件已存在且包含部分小节内容：
- 识别已完成的最后一个小节标题（`## X.Y`）
- 对照大纲确定下一个待写小节
- 向用户确认："已完成至 X.Y 节，接下来写 X.Z 节，是否继续？"

### Step 5: 撰写

**写作规范**：
- **章节编号格式**：统一使用阿拉伯数字编号，层级为 `1` → `1.1` → `1.1.1`。章标题用 `# 1　标题`，节标题用 `## 1.1　标题`，小节标题用 `### 1.1.1　标题`。不使用"第一章""一、""（一）"等中文序号。
- 论证结构：论点 → 论据（文献引用） → 分析 → 小结
- **脚注制**：每一处引用分配独立的脚注编号，文内用上标数字标注 `[1]`、`[2]`、`[3]`……编号按出现顺序递增，**同一文献在不同位置被引用时使用不同编号**
- 每段论述必须有文献支撑，引用来源必须是 library/ 中的实际内容
- **初稿优先直接引用**：在 draft 和 batch 模式撰写初稿时，引用文献内容应**优先采用直接引用**（加引号，从 chunk 逐字复制），而非用自己的话转述。直接引用便于用户在审阅阶段核对原文出处。将直接引用转化为更流畅的间接表述，留给后续 revise 模式处理。
- **直接引用必须逐字复制**：凡是加引号的直接引用（block quote 或行内引号），必须从 chunk 文件中**逐字逐句复制**原文，**严禁任何改写、概括或重组**。如需省略中间内容，用"……"标注省略位置。如果 chunk 原文有明显 OCR 错误，保留原文并用 `[sic]` 标注，或告知用户。间接引用（不加引号的转述）可以用自己的语言概括，但核心术语仍须与原文一致。
- **页码提取**：引用 chunk 内容时，找到所引文字最近的前置页码标记——格式为独占一行的 `[NNN]`（正则：`^\[(\d+)\]\s*$`）。将该数字作为脚注页码。如果所引内容跨两个标记（如 `[230]` 到 `[231]`），使用 `230-231`。如果 chunk 中无任何页码标记（常见于 EPUB 来源的英文文献），不标注页码——绝不可编造。
- 康德原文引用附德语原文，格式：`"中文引文"（"德语原文"）`
- **英文直接引用附中文翻译**：凡是直接引用英文文献原文的段落，在引文后紧跟括号附上中文翻译，格式：`"English quote"（中文翻译）`。短语级引用如果上下文已经清楚可酌情省略。
- 法律术语首次出现时给出定义或解释
- 段落之间有自然的逻辑过渡

**脚注格式（脚注制）**：

采用脚注制：每一处引用对应一条独立脚注，脚注使用 GB/T 7714 格式并标注具体页码。
引用信息从对应的 meta.md 或 chunk 文件的来源注释中获取。

**核心规则**：
- 每处引用 = 一个独立脚注编号 + 一条完整的引用条目（含页码）
- 同一文献被多次引用时，每次引用都是独立脚注，各标各的页码
- chunk 有页码标记 → 脚注末尾标注 `: 页码`（或 `: 起始页-结束页`）
- chunk 无页码标记 → 页码位置写 `[待补页码, 见 <chunk文件名>]`（如 `[待补页码, 见 chunk-03.md]`），绝不编造
- **不要**在正文中以 `(6:225)` 等形式行内标注科学院版页码，页码一律写在脚注中

示例（注意 [1] 和 [2] 引用同一本书的不同页码）：
```
正文：……它的简单性……当然一开始就必定使人惊奇。[1]
正文：……都被康德称为"自由的法则"[2]
正文：……应当就是《道德形而上学》这一体系[3]
正文：……将定言命令确立为唯一涉及道德法则的命令[4]
正文：……法权与强制之间存在分析性联系[5]

---
[1] 康德. 道德形而上学（注释本）[M]. 张荣, 李秋零, 译注. 北京: 中国人民大学出版社, 2013: 225.
[2] 康德. 道德形而上学（注释本）[M]. 张荣, 李秋零, 译注. 北京: 中国人民大学出版社, 2013: 214.
[3] 康德. 道德形而上学（注释本）[M]. 张荣, 李秋零, 译注. 北京: 中国人民大学出版社, 2013: 205.
[4] 康德. 道德形而上学的奠基[M]. 李秋零, 译. 北京: 中国人民大学出版社, 2013: 412-414.
[5] WILLASCHEK M. Right and Coercion[J]. International Journal of Philosophical Studies, 2009, 17(1): [待补页码, 见 chunk-03.md].
```

**保存**：将内容写入 `thesis/<章节文件名>.md`
文件命名由用户决定或根据大纲自动命名（如 `chapter1.md`、`introduction.md`）。

### Step 5.5: 暂停与报告（仅 draft 模式，batch 模式跳过此步）

写完当前小节后：
1. 将内容追加到 `thesis/<章节文件名>.md`
2. 向用户报告：
   - 已完成的小节名称和大致字数
   - 本节使用的主要文献及引用页码
   - 下一节的标题和预计内容概要（从大纲读取）
   - 如有文献不足之处，提示用户
3. **停下，等待用户回复**。用户可以：
   - "继续" → 写下一节
   - 提出修改意见 → 先修改当前节，改好后再继续
   - "跳过 X.Z" → 跳到更后面的节
   - "写完" / "一口气写完" → 切换为 batch 模式完成剩余节

### Step 6: 自检

撰写完成后自动检查：
- 所有引用标注是否在脚注中有对应条目
- 引用的观点是否与 library/ 中小块文件的原文一致
- 论证逻辑是否与大纲一致
- 与前后章节的衔接是否自然
- 字数是否在大纲预计范围内

将自检结果简要报告给用户。

## 并行写作模式（AI 主动判断触发）

**不需要用户记忆任何特殊指令。** 当用户的请求未指定具体某一章（如"开始写"、"写论文"、"写所有章节"）时，AI 主动检测状态并询问。

### 触发检测（在 Step 1 确定写作范围时执行）

若用户请求未明确指定单个章节：
1. 检查 `thesis/outline.md` 是否存在
2. 检查 `thesis/writing-plan.md` 是否存在
3. 检查 `thesis/` 目录下已有哪些章节文件（了解哪些章节已写、哪些待写）

根据检测结果：

**情况 A：outline 和 writing-plan 都存在，有多个待写章节**
→ 主动询问用户：
> 大纲和写作计划已就绪，有 N 章待写（列出章节名）。有两种写作方式：
> 1. **逐章写**：从某一章开始，每次写一章，写完你审阅后再继续
> 2. **并行写作**：同时启动所有章节（batch 模式），AI 并行生成完整初稿（速度快，适合先有个全局草稿再统一修改）
>
> 你倾向于哪种方式？或者指定从哪一章开始？

**情况 B：outline 存在但 writing-plan 不存在**
→ 告知用户写作计划尚未生成，建议先运行 `/thesis-outline`（或询问是否现在生成）

**情况 C：用户明确指定了某一章**（如"写第二章"）
→ 直接进入单章写作流程，不询问，不检测

### 并行写作执行流程

用户确认并行后：
1. **主上下文**读取 outline.md 和 writing-plan.md，确定待写章节列表
2. 为每个章节生成一个**独立的 Agent 子任务**，**并行启动**（在同一条消息中发出多个 Agent 调用）
3. 每个子 agent 的 prompt 包含：
   - 该章节的大纲内容（从 outline.md 摘取）
   - 该章节需要读取的 chunk 列表（从 writing-plan.md 摘取）
   - 前一章节的最后 2 段（用于衔接，如果是第一章则省略）
   - 完整的写作规范（Step 5 的全部要求）
   - 保存路径：`thesis/<章节文件名>.md`
4. **主上下文**等待所有子 agent 完成后：
   - 检查各章节之间的衔接是否自然（读取每章首尾各 2 段）
   - 检查各章脚注编号是否需要全文重编号（脚注制下每章可独立编号，也可全文连续）
   - 向用户报告完成情况和需要人工审查的问题

**子 agent 注意事项**：
- 每个子 agent 完整执行 Step 3 → Step 4 → Step 5 → Step 6
- 子 agent 内部脚注编号从 [1] 开始（每章独立编号）
- 绪论和结论建议在其他章节初稿完成后再写，可在询问用户时说明这一建议

## 写作质量标准

- 论点明确，每段有中心句
- 论证充分，避免无依据的断言
- 学术用语规范，避免口语化
- 逻辑严密，避免跳跃式推理
- 引用准确，所有引文可追溯到文献库
