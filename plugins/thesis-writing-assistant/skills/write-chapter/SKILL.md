# 章节撰写

撰写论文的具体章节内容。严格按以下流程执行，确保每段论述有文献支撑。

## 工作流程（严格按顺序执行）

### Step 1: 确定写作范围

解析用户请求，确定：
- 要写的章节/小节编号或名称
- 写作模式：
  - **draft**（默认）：生成初稿，覆盖完整章节结构
  - **expand**：在已有内容基础上扩展某一节
  - **revise**：根据用户反馈修改已有内容

### Step 2: 读取大纲

读取 `thesis/outline.md`，找到目标章节的大纲内容：
- 该章节的核心论点
- 在全文论证中的位置和作用
- 与前后章节的逻辑关系
- 大纲中标注的预计字数

如果大纲不存在，提示用户先运行 `/thesis-outline`。

### Step 3: 从文献库按需加载参考文献（关键步骤）

1. 读取 `library/index.md`
2. 根据当前章节的主题，从索引的描述中**筛选相关的小块**
3. **只读取**筛选出的小块文件（通常 2-5 个），不要读取无关的小块
4. 如果某个相关文献的 meta.md 有助于理解全局，也可以读取
5. 如果索引中找不到足够的相关文献，告知用户哪些方面的文献不足

**禁止**：一次性读取所有小块文件或整个 library/ 目录。

### Step 4: 读取已有内容

检查 `thesis/` 下是否已有该章节文件。
如有，读取现有内容以保持连贯性和避免重复。

也检查前一章节的结尾段落，确保衔接自然。

### Step 5: 撰写

**写作规范**：
- 每段 200-400 字为宜
- 论证结构：论点 → 论据（文献引用） → 分析 → 小结
- 引用格式：文内用上标数字标注，如 `[1]`、`[2]`
- 每段论述必须有文献支撑，引用来源必须是 library/ 中的实际内容
- 康德原文引用附德语原文，格式：`"中文引文"（"德语原文"）`
- 法律术语首次出现时给出定义或解释
- 段落之间有自然的逻辑过渡

**脚注格式**：
在章节末尾列出本章引用的参考文献，使用 GB/T 7714 格式。
引用信息从对应的 meta.md 或 chunk 文件的来源注释中获取。

**保存**：将内容写入 `thesis/<章节文件名>.md`
文件命名由用户决定或根据大纲自动命名（如 `chapter1.md`、`introduction.md`）。

### Step 6: 自检

撰写完成后自动检查：
- 所有引用标注是否在脚注中有对应条目
- 引用的观点是否与 library/ 中小块文件的原文一致
- 论证逻辑是否与大纲一致
- 与前后章节的衔接是否自然
- 字数是否在大纲预计范围内

将自检结果简要报告给用户。

## 写作质量标准

- 论点明确，每段有中心句
- 论证充分，避免无依据的断言
- 学术用语规范，避免口语化
- 逻辑严密，避免跳跃式推理
- 引用准确，所有引文可追溯到文献库
