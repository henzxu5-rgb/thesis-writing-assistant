# 大纲制定与写作计划

帮助用户确定论文的章节结构、论证逻辑和写作计划。

## 工作流程

### Step 1: 了解现状

- 检查 `thesis/outline.md` 是否已存在
- 读取 `library/index.md` 了解已有哪些文献资源；如需了解具体章节内容，再读该书的 `library/<书目目录>/index.md`
- 如果是调整（adjust），先读取现有大纲

### Step 2: 与用户对话确定方向

通过提问帮助用户明确：
1. **中心论点**：论文要论证什么？（如"康德的法治国家理念为现代法治提供了何种理论基础"）
2. **研究问题**：具体要回答哪些问题？
3. **研究方法**：文献分析、比较研究、历史考察等
4. **论文规模**：总字数要求、章节数量偏好
5. **导师反馈**：如果有导师的意见或要求，纳入考虑

### Step 3: 提出大纲方案

基于对话和已有文献，提出一个三级大纲（章 > 节 > 小节）：

- 每章标注预计字数
- 每节标注核心论点和主要依据的文献（从 index.md 中找到对应文献）
- 标注各章之间的逻辑关系

读取 `${CLAUDE_PLUGIN_ROOT}/skills/thesis-outline/references/outline-template.md`
参考法学论文的通用结构模式。

### Step 4: 论证逻辑检查

验证大纲的逻辑完整性：
- 中心论点是否贯穿始终
- 各章之间是否存在逻辑递进
- 是否有遗漏的关键论证环节
- 现有文献是否足以支撑各章论证（如有缺口，提示用户补充文献）

### Step 5: 保存大纲

将最终大纲保存到 `thesis/outline.md`。

大纲文件格式：
```markdown
# 论文大纲：<论文题目>

## 中心论点
<一段话阐述>

## 研究方法
<简述>

---

## 第一章 <章标题>（约 X 字）

### 1.1 <节标题>
- 核心论点：...
- 主要文献：...

### 1.2 <节标题>
- 核心论点：...
- 主要文献：...

<!-- 后续章节... -->

---

## 参考文献缺口
- <缺少某方面的文献，建议补充>
```

## 注意事项

- 不强制预设任何固定的章节结构
- 章节数量和划分完全取决于研究问题和用户意愿
- 大纲可以随时调整，用 `adjust` 模式修改
- 如果用户的研究还在早期阶段，可以先提出一个粗略的框架，后续细化
