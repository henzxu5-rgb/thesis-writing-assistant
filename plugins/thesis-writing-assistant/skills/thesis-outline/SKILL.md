# 大纲制定与写作计划

帮助用户确定论文的章节结构、论证逻辑和写作计划。

## 工作流程

### Step 1: 了解现状

- 检查 `thesis/outline/` 目录是否已存在（大纲采用分章存放：`outline/index.md` 是全局索引，`outline/introduction.md` / `outline/chapter1.md` / `outline/chapter2.md` / … / `outline/conclusion.md` 各自存放对应章节的详细规划）
- 读取 `library/index.md` 了解已有哪些文献资源；如需了解具体章节内容，再读该书的 `library/<书目目录>/index.md`
- 如果是调整（adjust），先读取现有大纲：至少读 `outline/index.md`，再按需读取要调整的那一章对应的分章文件（不必一次读完全部）

**旧格式 index.md 升级检测**：

读取现有 `outline/index.md` 后，检查是否缺少以下三节（这是 write-chapter 所依赖的全局骨架）：
- `## 各章的论证角色`
- `## 核心概念依赖链`
- `## 关键论证桥梁`

若任一缺失（常见于早期生成的"轻量索引"格式），主动告知用户：
> 检测到现有 index.md 使用旧格式，缺少 [各章论证角色 / 概念依赖链 / 论证桥梁] 三节中的 N 项。这些是 write-chapter 写作时的全局定位依据，缺失会导致单节写作与前后文脱节。是否现在升级？

升级流程：
1. 基于现有章节文件与中心论点，为每章撰写"论证角色"（强调"做什么工作"而非"写什么"）
2. 从各章节文件中提取核心概念，整理为依赖链（谁在哪章建立、谁在哪章被调用）
3. 梳理章与章、节与节之间的结构性转承关系，写入论证桥梁
4. 向用户确认后写回 index.md，保留原有"中心论点""研究方法""推进逻辑"等字段不变

若用户拒绝升级，告知用户 write-chapter 运行时可能需要用户手动补充上下文信息。

### Step 2: 与用户对话确定方向

通过提问帮助用户明确：
1. **中心论点**：论文要论证什么？（如"康德的法治国家理念为现代法治提供了何种理论基础"）
2. **研究问题**：具体要回答哪些问题？
3. **研究方法**：文献分析、比较研究、历史考察等
4. **论文规模**：总字数要求、章节数量偏好
5. **导师反馈**：如果有导师的意见或要求，纳入考虑

### Step 3: 提出大纲方案

基于对话和已有文献，提出一个三级大纲（章 > 节 > 小节）：

- 每章标注预计字数
- 每节标注核心论点和主要依据的文献（从 index.md 中找到对应文献）
- 标注各章之间的逻辑关系

读取 `${CLAUDE_PLUGIN_ROOT}/skills/thesis-outline/references/outline-template.md`
参考法学论文的通用结构模式。

### Step 4: 论证逻辑检查

验证大纲的逻辑完整性：
- 中心论点是否贯穿始终
- 各章之间是否存在逻辑递进
- 是否有遗漏的关键论证环节
- 现有文献是否足以支撑各章论证（如有缺口，提示用户补充文献）

### Step 5: 生成分章存放的大纲—写作计划文件

将大纲、论证思路、写法指令和文献映射**按章拆分存放**在 `thesis/outline/` 目录下。
此目录是用户掌控全局的唯一入口，也是 `write-chapter` 读取的唯一规划来源。

**目录结构**：
```
thesis/outline/
├── index.md          # 全局索引：中心论点、研究方法、推进逻辑、章节文件列表、参考文献缺口、写作状态
├── introduction.md   # 绪论详细规划
├── chapter1.md       # 第一章详细规划
├── chapter2.md       # 第二章详细规划
├── chapter3.md       # 第三章详细规划
└── conclusion.md     # 结论详细规划
```

> 拆分的目的：单一大纲文件可能长达数百行，每次 `write-chapter` 写单章时都要加载全部内容，既浪费 token 也淹没上下文。按章拆分后，写某一章时只读 `outline/<chapter>.md` 即可。

**生成流程**：
1. 读取 `library/index.md`，逐个读取每本书的 `library/<name>/index.md`
2. 对照大纲中每章每节的核心论点，从 chunk 描述和子话题中筛选相关 chunk
3. 为每个小节同时填写：核心论点、论证思路、写法、文献块列表
4. 先写 `outline/index.md`（全局索引），再逐一写各章文件

**index.md 格式**（全局骨架，写任何一节都必读）：

> **设计原则**：index.md 是"论证骨架"，不是"轻量索引"。它必须让 AI 在写任何一节时都能快速定位"我这一节在全文论证链条中承担什么角色"。执行细节（chunk 列表、写法、论证思路等）留在各章文件中，但**全局论证逻辑必须在此完整呈现**，否则 AI 写单节时容易迷失在局部细节里，与前文或后续章节脱节。

```markdown
# 论文大纲索引：<论文题目>

> 本索引是全局论证骨架，写任何一节前都应先读本文件，再读对应章的详细规划。

## 章节文件

| 章节 | 文件 | 字数规模 |
|---|---|---|
| 绪论 | [introduction.md](introduction.md) | — |
| 第一章　<标题> | [chapter1.md](chapter1.md) | 约 X 字 |
| …  | … | … |
| 结论 | [conclusion.md](conclusion.md) | — |

## 中心论点
<一段话完整阐述论文要论证的命题与其独特性>

## 研究方法
<简述文本分析、比较研究、概念分析等具体方法>

## 各章的论证角色

> 重点不是"这一章写了什么"，而是"这一章在全文论证链条中做什么工作"。

- **绪论**：<提出问题 / 综述现状 / 说明方法与结构>
- **第一章**：<例：通过检验两种错误立场的失败，反向提炼正确答案必须满足的三个条件>
- **第二章**：<例：建立"具体化"立场，逐一正面验证三个条件>
- **第三章**：<例：从概念层转入制度层，通过双重归谬论证具体化立场的不可替代性>
- **结论**：<收束 / 理论意义 / 研究局限>

## 核心概念依赖链

> 哪些概念在哪章建立、被后续哪章使用。AI 写后续章节时要知道前面哪些概念已经铺好，可以直接调用而不必重新定义。

- **<概念 A>**：第 X 章引入并定型 → 第 Y 章应用 → 第 Z 章制度化
- **<概念 B>**：第 X 章与 <概念 A> 对举 → 贯穿全文
- **<术语 C>** 的译名/定义：<在哪一节首次定型，后续统一沿用>

## 关键论证桥梁

> 章与章之间、节与节之间的逻辑接榫点。这是最容易被单节写作忽略、需要 AI 显式知道的部分。

- **第 X 章 → 第 Y 章**：<前者的 X 结论 = 后者的 Y 起点，转承方式>
- **X.2 ↔ X.3**：<镜像结构 / 并列 / 递进，二者如何呼应>
- **X 节 → X+1 节**：<从消极面转入积极面，或其他结构性转折>

## 推进逻辑

<一段话串联各章的论证走向，使用箭头串起关键论证步骤，作为上述内容的一句话总括>

## 参考文献缺口
- <某节缺少足够的文献支撑，建议补充>

## 写作状态
| 章节 | 文件 | 状态 |
|---|---|---|
| … | … | 待写 / 进行中 / 已完成 |
```

**关于"各章论证角色 / 概念依赖链 / 关键论证桥梁"三项的写作要点**：
- **论证角色** 用动词主导（"提炼条件""验证立场""归谬论证"），避免只描述内容
- **概念依赖链** 标明每个核心概念的"生命周期"——在哪诞生、在哪被调用、在哪被升级
- **论证桥梁** 显式写出章/节之间的结构关系（镜像 / 递进 / 转折 / 铺垫），这是 AI 独立写单节时最容易丢失的信息

**章文件格式**（`outline/chapter1.md` 等，内含该章所有小节的四字段详情）：
```markdown
# 第一章　<章标题>（约 X 字）

**本章任务**：<一句话概括该章在全文中的角色>

**章引言**：<承上启下的段落要点>

## 1.1　<节标题>

### 1.1.1　<小节标题>

- **核心论点**：<这一小节要论证什么，1-2 句>
- **论证思路**：<先论证 A，再用文献 X 反驳 B，最后得出 C；
  说清楚论证的步骤和逻辑展开方式>
- **写法**：<以何种方式呈现——学术综述 / 文本细读 / 比较研究 /
  概念分析等；语气和侧重点的指示>
- **文献**：
  - `guyer-2005/chunk-42.md` — <该 chunk 与本节的关联说明>
  - `ripstein-2009/chunk-05.md` — <关联说明>

### 1.1.2　<小节标题>
- **核心论点**：...
- **论证思路**：...
- **写法**：...
- **文献**：
  - `kant-metaphysics/chunk-12.md` — <关联说明>

## 1.2　<节标题>
<!-- … -->
```

**注意**：
- 四个字段（核心论点 / 论证思路 / 写法 / 文献）缺一不可——这是用户掌控写作走向的关键
- 每个 chunk 条目附一句话关联说明，让用户能直接判断该 chunk 的作用
- 如果某小节的核心论点在文献库中找不到对应 chunk，**不要编造**，在 `index.md` 的"参考文献缺口"中列出
- 同一个 chunk 可以被多个章节引用
- 用户随时可以手动编辑任一章文件来调整其中任何小节的论证思路、写法或文献列表，`write-chapter` 下次写作时自动遵循修改后的内容
- 如果中心论点或推进逻辑发生变化，同步更新 `index.md`

## 注意事项

- 不强制预设任何固定的章节结构
- 章节数量和划分完全取决于研究问题和用户意愿
- 大纲可以随时调整，用 `adjust` 模式修改；调整后所有四个字段（核心论点/论证思路/写法/文献）都应同步更新
- 如果用户的研究还在早期阶段，可以先提出一个粗略的框架，后续细化
