# 文献搜索与下载

自动搜索学术文献的可下载版本，下载 PDF 等原始文件并存入 `sources/` 目录。

## 核心原则

- **不突破付费墙**：遇到付费文献只记录链接，提示用户自行获取
- **不自动读取 sources/**：下载的文件仅存储，不纳入 AI 的文献检索流程
- **诚实报告**：如实汇报每篇文献的下载结果，不伪造下载状态

---

## 工作流程

### Step 1: 确定目标文献

解析用户的下载请求：

- **用户指定文献**：直接使用用户提供的标题、作者、引用信息
- **`--list` 模式**：读取 `research/` 目录下 collect-references 保存的推荐列表，提取其中的文献条目
- **交互确认**：如果信息不足以精确定位文献，询问用户补充

**去重检查**：读取 `sources/index.md`，跳过已下载的文献（除非用户明确要求重新下载）。

### Step 2: 搜索下载源

对每篇目标文献，使用 WebSearch 搜索可下载版本。

**搜索策略（按优先级）：**

**外文文献：**
1. Google Scholar — 搜索标题 + 作者，寻找 [PDF] 标记的直接链接
2. ResearchGate / Academia.edu — 作者自行上传的全文
3. 大学机构仓库 — 开放获取的学位论文和预印本
4. PhilPapers — 哲学类文献专用（康德研究重点来源）
5. CORE / Unpaywall — 开放获取聚合平台
6. 出版商网站 — 作为最后手段，可能需要付费

**中文文献：**
1. 百度学术 — 搜索标题，寻找免费下载链接
2. 知网开放获取 — CNKI 部分免费论文
3. 万方数据 — 部分免费获取
4. 各大学机构仓库 — 学位论文
5. 国家哲学社会科学文献中心 — 部分免费学术资源

**搜索关键词构造：**
- 优先使用 DOI（如有）：直接搜索 `doi:10.xxxx/xxxxx filetype:pdf`
- 使用精确标题搜索：`"论文完整标题" filetype:pdf`
- 补充作者名：`作者姓名 "论文标题" pdf`
- 如果 collect-references 提供了下载线索（DOI/URL），优先使用

### Step 3: 下载文件

对每篇找到下载链接的文献：

1. 确保目录存在：`mkdir -p sources/`
2. 生成文件名（规则见下方）
3. 使用 Bash 执行下载：
   ```bash
   curl -L -o "sources/<文件名>.pdf" "<下载URL>"
   ```
4. 同一域名的连续下载之间间隔 2 秒，避免触发限流

**文件命名规则：**
- 英文：`作者姓-年份-简短标题.pdf`（如 `ripstein-2009-force-freedom.pdf`）
- 中文：`作者拼音-年份-简短标题.pdf`（如 `deng-xiaomang-2004-chunli-pipan.pdf`）
- 简短标题取 2-4 个关键词，用连字符连接，全小写
- 避免特殊字符，只使用字母、数字和连字符

### Step 4: 验证文件

对每个下载的文件进行验证：

```bash
file "sources/<文件名>.pdf"
```

- 如果输出包含 `PDF document`：下载成功
- 如果输出为 `HTML document` 或 `ASCII text`：可能下载了登录页面或错误页面，删除文件并标记为失败
- 如果文件大小为 0：删除并标记失败

```bash
ls -la "sources/<文件名>.pdf"
```

检查文件大小是否合理（学术 PDF 通常 100KB - 50MB）。

### Step 5: 更新追踪索引

在 `sources/index.md` 的表格中追加记录：

| 文献 | 文件名 | 状态 | 下载日期 | 来源 | 已处理 |
|------|--------|------|----------|------|--------|
| 完整引用（GB/T 7714） | 文件名.pdf | 已下载 | 2026-04-02 | URL | 否 |

**状态值：**
- `已下载` — 文件已成功保存到 sources/
- `仅链接` — 找到了文献但需付费/登录，记录了 URL
- `未找到` — 未能在网上找到该文献的可获取版本

### Step 6: 结果报告

向用户展示下载结果汇总：

**成功下载的文献：**
列出文件名和存储位置。

**需要手动获取的文献（付费/需登录）：**
- 提供找到的最佳 URL
- 建议获取途径：
  - 通过学校 VPN 访问数据库
  - 通过图书馆电子资源平台
  - 馆际互借服务

**未找到的文献：**
- 确认文献信息是否准确
- 建议替代版本或相关文献

**后续步骤提醒：**
> 下载的文件存放在 `sources/` 目录。要让 AI 能使用这些文献：
> 1. 将 PDF 转换为 Markdown 格式
> 2. 放入 `inbox/` 目录
> 3. 运行 `/process-reference` 进行切分和索引

---

## 注意事项

- **绝不尝试绕过付费墙或访问限制**
- 下载失败时不重试超过 1 次
- 如果单个文件超过 50MB，先告知用户再下载
- 不读取或解析下载的 PDF 内容 — 此目录不属于 AI 的文献检索范围
- 优先下载开放获取版本，即使存在付费的正式出版版本
