# 文献处理与切分

将用户提供的完整文献（整本书或论文的 MD 文件）自动切分成小块，
为每块生成描述摘要，并更新全局索引，以便后续按需检索。

## 工作流程

### Step 1: 定位原始文献

扫描 `${CLAUDE_PLUGIN_ROOT}/inbox/` 目录，列出所有待处理的 MD 文件。
如果用户指定了具体文件名，直接使用该文件。

如果 inbox/ 为空且用户提供了文件路径，从该路径读取。

### Step 2: 读取并分析文献结构

读取原始文献内容，分析：
- 文献类型：专著(M)、论文(J)、译著、学位论文(D)等
- 文档结构：标题层级（`#`, `##`, `###`）、章节划分、段落组织
- 总字数和预计切分数量
- 引用信息（作者、书名/篇名、出版信息）——通常在文件开头或由用户提供

向用户简要报告分析结果，确认切分方案后继续。

### Step 3: 确定切分方案

**切分原则**：
- 每块目标 **500-1500 字**（中文字符计）
- 按**逻辑单元**切分，不在论证中间切断
- 保持每块的**主题完整性**——一个块应该讨论一个相对独立的主题

**整本书**（用户可能已按章预分为多个 MD）：
- 如果用户已按章分好多个 MD → 将每个 MD 视为一个大块，按节/主题进一步切分
- 如果是一个大 MD → 按章节标题自动识别边界
- 优先以 `##`（二级标题）为切分边界
- 如果某节超过 1500 字，在 `###`（三级标题）或自然段落组处进一步切分
- 如果某节不足 500 字，与相邻小节合并

**单篇论文**：
- 按逻辑段落切分：摘要、引言、各节、结论
- 每节如果超过 1500 字，按子标题或自然段落组进一步切分
- 摘要单独一块（即使很短，因为信息密度高）

### Step 4: 执行切分并保存

1. 为该文献创建目录：`${CLAUDE_PLUGIN_ROOT}/library/<文献简称>/`
   - 文献简称使用英文 kebab-case，如 `kant-metaphysics-morals`、`zhang-rechtstaat-2020`
   - 如果用户通过 `--name` 指定了简称，使用用户指定的

2. 生成 `meta.md`：
```markdown
# <文献标题>

## 引用信息
<GB/T 7714 格式的完整引用>

## 全书/全文概述（200-300字）
<概括文献的核心论点、研究方法、主要结论>

## 与论文的关联
<简述该文献对论文写作的价值和可能的用途>

## 小块清单
共 N 个小块，详见 library/index.md
```

3. 逐块保存为 `chunk-01.md`、`chunk-02.md`...：
   - 每个 chunk 文件开头加一行注释标明来源和位置：
```markdown
<!-- 来源：<文献标题> | 位置：<章节信息> -->

<原文内容>
```

### Step 5: 为每块生成描述

为每个 chunk 写一句简洁但信息丰富的描述（40-80字），包含：
- 该块讨论的**核心主题**
- **关键概念或术语**（便于后续检索匹配）
- 在全文中的**位置/语境**

好的描述示例：
- "导论——法权学说与道德学说的区分，Recht 概念定义，外在立法与内在立法的区别"
- "第三章·公法——国家的三权分立原则，立法权、行政权、司法权的康德式论证"
- "文献综述——国内康德法哲学研究的三个阶段（1980s引介、2000s深化、2010s反思）"

差的描述示例（太模糊，无法判断是否需要读取）：
- "第一章的内容"
- "关于康德的讨论"
- "一些论述"

### Step 6: 更新全局索引

读取 `${CLAUDE_PLUGIN_ROOT}/library/index.md`，在末尾追加新文献的索引条目。

格式：
```markdown
## <文献标题>（`<目录名>/`）
- **meta.md**: [文献类型标记] <简要引用信息>。<全文一句话概述>
- **chunk-01.md**: <描述>
- **chunk-02.md**: <描述>
- ...
```

文献类型标记：`[M]` 专著、`[J]` 期刊论文、`[D]` 学位论文、`[C]` 会议论文、`[N]` 报纸

### Step 7: 清理

将处理完的原始文件从 `inbox/` 移到 `inbox/processed/`（自动创建该目录）。

向用户报告处理结果：
- 文献名称
- 切分为几个小块
- 索引已更新

## 批量处理

如果 `inbox/` 中有多个文件，逐个处理，每处理完一个向用户报告。

## 特殊情况

- **用户已按章分好多个 MD**：将它们视为同一文献的多个部分，放入同一个 `library/<name>/` 目录，chunk 编号连续
- **文献缺少引用信息**：询问用户补充，或标注 `[引用信息待补充]`
- **文献内容过短**（不足 500 字）：不切分，整体作为一个 chunk，仍然生成 meta.md 和索引条目
